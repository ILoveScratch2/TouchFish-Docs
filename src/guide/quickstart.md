# 快速开始

这个教程将帮助您在 5 分钟内启动并运行 TouchFish 聊天系统。

## 前置准备

在开始之前，请确保：

- ✅ 已经[下载](./download.md)对应平台的 TouchFish 版本
- ✅ 确保您的设备在同一局域网内
- ✅ 关闭防火墙或开放对应端口（可选）

---

## 第一步：启动服务器

服务器端是整个聊天系统的核心，一个局域网内只需要一个服务器。

如果您只是想加入别人的聊天室，可以跳过这一步。

### Windows 用户

1. 双击运行 `chat.exe`
2. 按提示输入配置信息

### Linux/macOS 用户

```bash
# 使用预编译版本
./chat

# 或使用 Python 源码
python chat.py
```

### 配置服务器

启动后，程序会要求您输入以下信息：

```text
Connect to IP: 192.168.1.100        # 输入本机 IP 或 0.0.0.0
The maximum times of connecting: 10  # 最大连接数
The connecting port (must be spare): 8080  # 端口号（确保未被占用）
```

::: info 如何获取本机 IP？
**Windows**：打开命令提示符，输入 `ipconfig`，查找"IPv4 地址"

**Linux/macOS**：打开终端，输入 `ip a` 或 `ifconfig`
:::

::: warning 端口占用检查
**Windows**：`netstat -an | findstr 8080`

**Linux/macOS**：`netstat -an | grep 8080`

如果有输出，说明端口被占用，请更换端口号。
:::

### 启动成功

看到类似以下输出即表示服务器启动成功：

```text
TouchFish(Server) started successfully, 192.168.1.100:8080.
当前版本：v3.2.0，最新版本：v3.2.0
192.168.1.100:8080>
```

::: tip 记下这些信息
将 **IP 地址**（`192.168.1.100`）和 **端口号**（`8080`）告诉其他用户，他们需要这些信息来连接。
:::

---

## 第二步：客户端连接

### 使用 LTS 客户端

1. 双击运行 `client.exe` 或 `./client`
2. 在连接窗口输入服务器信息：
   - 服务器 IP：`192.168.1.100`
   - 端口：`8080`
   - 用户名：您的昵称
3. 点击"连接"

其他客户端类似操作，待补充...

### 等待审批

::: info 连接流程
1. 客户端发送连接请求
2. 服务器收到请求（根据配置可能自动通过或需要手动审批）
3. 连接成功后即可开始聊天
:::

---

## 第三步：开始聊天

### 发送消息

连接成功后，您可以：

1. 在输入框中输入消息
2. 点击"发送"按钮
3. 消息会立即发送给所有在线用户


::: warning 文件大小限制
文件传输可能受到服务器配置限制，默认情况下建议文件不超过 100MB。
:::

---

## 第四步：服务器管理（可选）

如果您是服务器管理员，可以使用命令来管理聊天室。

### 在服务器控制台输入命令

在服务器窗口的提示符 `192.168.1.100:8080>` 后输入命令：

```bash
# 查看在线用户
search online

# 广播消息
broadcast 欢迎来到 TouchFish 聊天室！

# 查看帮助
help
```

### 使用管理员客户端

如果需要远程管理，可以使用 `admin.exe` 或 `admin`：

1. 在服务器端开启管理员模式：
   ```bash
   admin on
   ```
2. 添加管理员 IP：
   ```bash
   admin add 192.168.1.101
   ```
3. 在管理员机器上运行 `admin.exe`，输入服务器 IP 和管理端口

详细使用方法请参见[管理员使用指南](./admin.md)。

## 常见问题

### 无法连接服务器？

1. **检查 IP 和端口**：确保输入正确
2. **检查网络**：确保在同一局域网内
3. **检查防火墙**：暂时关闭防火墙测试
4. **检查端口占用**：确保端口未被其他程序占用

### 连接后没有反应？

1. **等待审批**：如果服务器开启了手动审批，需要等待管理员同意
2. **查看服务器**：在服务器控制台输入 `req` 查看待审批请求
3. **接受请求**：输入 `accept 0` 接受第一个请求

### 消息发送失败？

1. **检查连接状态**：确保仍然在线
2. **消息长度限制**：单条消息不要太长
3. **敏感词过滤**：消息可能包含被过滤的词
4. **某些服务器（如bopid）禁止发送文件**：请联系管理员了解详情
