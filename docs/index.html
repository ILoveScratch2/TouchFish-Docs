协议概览

TouchFish 使用 JSON over TCP 进行通信。

主要消息类型

  1. JOIN - 加入流程

    • JOIN.REQUEST - 客户端请求加入
    • JOIN.RESPONSE.ACCEPTED - 服务器接受
    • JOIN.RESPONSE.REJECTED - 服务器拒绝

  2. CHAT - 聊天消息

    • CHAT.SEND - 发送消息
    • CHAT.RECEIVE - 接收消息
    • CHAT.SENDING_RESULT - 发送结果

  3. FILE_TRANSFER - 文件传输

    • FILE_TRANSFER.REQUEST - 请求发送文件
    • FILE_TRANSFER.ANNOUNCE - 通知文件可下载
    • FILE_TRANSFER.TRANSFERRING.* - 传输过程

  4. MISC - 杂项

    • MISC.BROADCAST - 广播消息
    • MISC.USERS - 用户列表
    • MISC.KICK - 踢出用户

详细协议规范请查看通信协议文档。

使用文档的技巧

快速查找

使用浏览器的搜索功能（Ctrl + F 或 Cmd + F）快速定位：

  • 搜索命令名称：如 broadcast
  • 搜索配置项：如 ENTER_AFTER_PROMISE
  • 搜索消息类型：如 CHAT.SEND

命令格式说明

文档中使用以下格式表示命令：

  • <必需参数>：必须提供的参数
  • [可选参数]：可以省略的参数
  • 选项1 | 选项2：多选一的参数
  • ...：可以重复的参数

示例：

ban ip add <IP1> [IP2] [IP3] ...

表示：

  • ban ip add 是固定的命令部分
  • <IP1> 是必需的，至少要提供一个 IP
  • [IP2] [IP3] ... 是可选的，可以提供多个 IP

代码示例

文档中的代码示例可以直接复制使用：

# 这是注释
search online        # 查看在线用户
broadcast 欢迎！     # 广播欢迎消息

配置示例

JSON 配置示例都是完整且有效的：

{
  "ban": {
    "words": ["test"],
    "ip": [],
    "length": 500
  },
  "ENTER_AFTER_PROMISE": false
}

常见任务

任务 1：设置审批模式

需求：新用户加入需要审批

命令：

set ENTER_AFTER_PROMISE on

配置文件：

{
  "ENTER_AFTER_PROMISE": true
}

参考文档：

  • 配置文件 - ENTER_AFTER_PROMISE
  • 服务器命令 - set

任务 2：封禁用户 IP

需求：封禁恶意用户的 IP 地址

命令：

ban ip add 192.168.1.110

配置文件：

{
  "ban": {
    "ip": ["192.168.1.110"]
  }
}

参考文档：

  • 服务器命令 - ban
  • 配置文件 - ban.ip

任务 3：添加敏感词

需求：过滤不当内容

命令：

ban words add 脏话 广告

配置文件：

{
  "ban": {
    "words": ["脏话", "广告"]
  }
}

参考文档：

  • 服务器命令 - ban
  • 配置文件 - ban.words

任务 4：开启远程管理

需求：允许远程管理服务器

命令：

admin on
admin add 192.168.1.120

参考文档：

  • 服务器命令 - admin
  • 管理员使用教程
